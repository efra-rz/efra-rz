Primeros pasos con git

Post

Historia

Git es la tecnología de control de versiones desarrollada por Linus Torvalds. Comenta Torvalds en la sesión "Code Power" de la conferencia TED2016 que uno de los temas que surgieron para mantener Linux fue cuando paso a tener mucha gente trabajando en el proyecto y cómo el tenía que mantener ese código, esa fue la principal razón del nacimiento de Git. "Git es mi segundo gran poryecto después de linux, y su razón de exisitir es para poder mantener Linux".

Funcionamiento

Imagina la que tienes una cámara de fotos mágica a travéz de la cuál tu puedes viajar en el tiempo entre las fotos que sacas, de modo que puedes corregir tus errores o probar desiciones como "que pasaría si" sin tomar ningún riesgo, puesto que podrías tomar una foto en un punto seguro de tu historia para volver.

Pues está es una de las muchas acciones que puedes hacer con el sistema de control de versiones Git, crear "Checkpoints" de tus proyectos y viajar entre ellos. Para ello es seguir los estos pasos

  • Modificas una serie de archivos en tu directorio de trabajo.
  • Añades estos archivos en un espacio denominado "área de preparación".
  • Confirmas cambios, esto hace que los archivos salten del "área de preparación" a una "instantánea" o punto en el cuál ya estás listo para saltar en el espacio tiempo.

Instalación

Podemos trabajar con Git tanto en un entorno gráfico cómo en la línea de comandos, a mi punto de vista recomendaría empezar con la línea de comandos, creo que la curva de aprendizaje puede ser mucho más sencilla porque con ella sólo dedicas tu atención a la acción específica a realizar.

Es muy probable que ya venga instalado tanto en Linux como para Mac en caso de no ser así lo puedes descargar directo de la página de Git, en Windows no biene instalado por defecto ahy si tienes que descargarlo, te dejo los Links a continuación:

  • Descarga para Mac
  • Descarga para Linux
  • Descarga para Windows

Configuración

Una vez Git instalado comprobamos la versión de éste con el siguiente comando:

git --version

Ahora pasamos a configurar tu nombre de usuario correo y un el editor de texto que desees utilizar.

git config --global user.name "John Doe"

git config --global user.email johndoe@example.com

git config --global core.editor nano

git config --list

Comandos Básicos del flujo de trabajo

Primero abrimos la terminal en el directorio de trabajo del proyecto y ejecutamos el comando:

git init

Con este comando agregamos todos nuestros archvios al área de preparación:

git add .

Con este comando revisamos el estatus de nuestro proyecto:

git status

Con éste comando agregamos creamos el commit o instantánea de nuestro proyecto:

git commit -m "Iniciamos el proyecto"

En este punto ya está listo nuestro punto en la historia para viajar, si deseamos crear nuevos puntos en la historia volvemos a repetir los pasos de agregar archivos al área de preparación, revisar cambios y hacemos el commit o instantánea.

Si llegaste hasta este punto de este artículo agradezco mucho tu tiempo, en el siguiente artículo veremos más en forma con un caso práctico el funcionamiento de esta genial herramienta.

¡Saludos y se cuidan!
